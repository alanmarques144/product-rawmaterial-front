# Challenge - Production Suggestion Front-end

This repository contains the Front-end application for the Challenge Production Module, developed as a technical assessment. It provides a responsive, modern user interface to manage raw materials, configure product recipes (Bill of Materials), and view optimal production plans generated by the back-end algorithm.

## 🎯 Project Overview & Requirements Met

This project was built to fulfill the following requirements:
* **Decoupled Architecture (RNF002):** A standalone SPA (Single Page Application) consuming a REST API.
* **Responsiveness (RNF003):** Fully responsive layout adapting to different screen sizes using Material-UI.
* **Frameworks (RNF006):** Built with **React** (via Vite for fast bundling) and **Redux Toolkit** for global state management.
* **Language (RNF007):** Code, variables, and documentation are written in English.
* **Functionality (RF005 - RF008):** Complete GUI for managing Raw Materials, Products, dynamic Material Associations (BOM), and viewing Production Suggestions.
* **Testing (Desirable):** End-to-End (E2E) testing implemented using **Cypress**.

## 🛠️ Technologies & Stack

* **React 18** (Bootstrapped with Vite)
* **Redux Toolkit (RTK)** (State management and async Thunks)
* **Material-UI (MUI)** (Component library, custom theming, and responsive layout)
* **Axios** (HTTP Client for API communication)
* **React Router v6** (Client-side routing)
* **Cypress** (E2E Integration Testing)

## 🏗️ Project Structure

The project follows a clean, feature-oriented structure:
* `cypress/e2e/`: Contains the End-to-End automated tests.
* `src/components/`: Reusable UI components (e.g., Layout, Navigation Drawer, App Bar).
* `src/pages/`: Main application views (`RawMaterials`, `Products`, `Production`).
* `src/services/`: API integration layer using Axios (`api.js`, `productService.js`, etc.).
* `src/store/`: Redux configuration, separating features into specific slices (`productSlice.js`, etc.).
* `src/theme.js`: Custom MUI theme configuration to match the requested corporate identity.

## ⚙️ How to Run the Project

### Prerequisites
* **Node.js** (v18 or higher recommended)
* Make sure the **Quarkus Back-end API** is running on `http://localhost:8080`.

### Step 1: Install Dependencies
Open your terminal in the project root directory and run:
```bash
npm install
```
### Step 2: Start the Development Server
Run the Vite development server:
```bash
npm run dev
```
The application will be available at `http://localhost:5173`.

## 🧪 Running Automated Tests (Cypress)

The project includes E2E integration tests simulating a user navigating through the interface, opening modals, and interacting with the system.

With both the Back-end and Front-end servers running, open a new terminal and choose one of the following commands:

**To open the Cypress GUI (Interactive Mode):**
```bash
npm run cypress:open
```
Select "E2E Testing", choose your preferred browser, and run the `product_rawmaterial.cy.js` spec.

**To run the tests in Headless Mode (Terminal only):**
```bash
npm run cypress:run
```

## 🎨 Design Decisions

- **Layout:** A persistent lateral Navigation Drawer and a top App Bar were implemented following Material Design guidelines, ensuring quick access to all functional requirements.

- **Dynamic Forms:** The Product creation form allows users to dynamically add or remove multiple raw materials in a single step, improving UX and fulfilling RF007 elegantly.

- **Global State:** Redux Toolkit was used to cache API responses, reducing redundant network requests and maintaining a single source of truth for the UI state.
